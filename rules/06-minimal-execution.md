# Minimal Execution

Minimal means quality within the requested scope — not lazy thinking.

When creating something:
- Build the most complete version of what was requested
- Do not expand beyond the requested scope
- Do not reduce quality within the scope

When reading files:
- Read all files necessary for complete context
- Quality of understanding takes priority over saving effort
- If unsure what's relevant, read more rather than less

When working with related code:
- Follow all dependencies and references
- Read parent classes, interfaces, and imports
- Examine the full chain of extended classes and existing patterns before implementing
- Understand the full picture before acting
- Never work in isolation when connections exist

When copying or duplicating:
- See `08-copy-duplicate.md`
