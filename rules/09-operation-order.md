# Operation Order

When moving or reorganizing code or files:
- Try to move and rename existing items first
- If moving is not possible: create new, then delete old
- Never delete first and create after

When refactoring or restructuring:
- Preserve existing code while transforming
- Implement the new structure before removing the old one
- Ensure continuity throughout the process

Priority order:
1. Move/rename existing items (preferred)
2. Create new, then remove old (alternative)
3. Never: delete first, create after
