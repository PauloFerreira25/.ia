# Concurrency Examples

When demonstrating concurrent or parallel execution scenarios:
- Use clear sequential task numbering
- Never use generic "1, 2, 3 run concurrently" without specific context
- Show explicitly the execution order and timing

Use this structure:
```
Task 1: [specific action with timing]
Task 2: [specific action with timing]
Task 3: [specific action with timing]
Result: [what happens and when]
```

When showing parallel execution:
- Number each step sequentially
- Describe exactly what each task does
- Indicate timing or ordering relationships
- Clearly demonstrate the resulting problem or advantage
